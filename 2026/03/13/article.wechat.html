从 3 月 7 日到 3 月 13 日，OpenClaw 连着发了 5 个版本。作为一个天天用的人，我挑了几个真正影响使用体验的更新聊聊。

更新有多快？

版本发布时间类型
2026.3.83 月 9 日正式版
2026.3.11-beta.13 月 12 日测试版
2026.3.113 月 12 日正式版
2026.3.123 月 13 日正式版

一天一个版本，这节奏放在开源项目里也是没谁了。

这 5 个更新最值得看

1. Dashboard v2 全新改版（3.12）

这次 UI 改动最大。以前那个控制台说实话有点程序员审美，现在完全不一样了。

多了什么：

  • 模块化设计：概览、聊天、配置、Agent、会话五个视图
  • 命令面板（Command Palette）：按个快捷键就能跳转，省得鼠标点来点去
  • 移动端底部标签栏：手机终于能正常用了
  • 聊天工具增强：斜杠命令、搜索、导出、置顶消息

实际体验：

  • 新手不用看文档也能摸索出来
  • 老用户效率确实高了，尤其是那个命令面板
  • 手机端从"能用"变成"好用"

2. Fast Mode 快速模式（3.12）

OpenAI 和 Claude 都支持了快速通道。

用法很简单：

/fast

支持的地方：

  • 终端界面（TUI）
  • 网页控制台（Control UI）
  • Agent 控制协议（ACP）

感受：

  • 响应确实快了，虽然官方没给具体数字
  • 适合需要反复调试的场景
  • 可以按会话开，不用全局启用，成本控制更灵活

3. 安全修复一大堆（3.11-3.12）

这次安全更新特别多，我数了一下有 10 几个。挑几个重要的说：

  • 设备配对：`/pair` 和 `openclaw qr` 改用短生命周期 token，不会在聊天或二维码里暴露网关凭证了
  • 插件安全：克隆的仓库不会自动执行代码，需要显式信任
  • 执行审批：处理了零宽度 Unicode 字符的欺骗问题
  • 网关认证：防止设备共享 token 被滥用

这说明什么？

用 OpenClaw 的人越来越多了，而且开始用在正经的生产环境里。安全不再是"以后再说"的事。

4. Kubernetes 官方支持（3.12）

企业用户应该会很开心。

给了什么：

  • 原生 K8s 安装路径
  • Raw manifests
  • Kind 配置
  • 完整部署文档

适合谁：

  • 团队规模部署
  • 需要高可用的场景
  • 已经在用云原生基础设施的

5. sessions_yield（3.12）

这个功能比较硬核，面向做复杂编排的用户。

能干什么：

  • 编排器可以立即结束当前回合
  • 跳过排队的工具工作
  • 带着隐藏数据进入下一轮

什么时候用：

  • 多步骤任务需要中途改变策略
  • 子 Agent 之间要传递状态
  • 长流程需要灵活控制

移动端终于能用了

iOS（3.11）

  • 加了欢迎页面，Agent 状态实时显示
  • 修复了后台切回前台的断连问题

macOS（3.11-3.12）

  • 聊天界面可以选模型了
  • 思考级别设置会记住
  • Ollama 配置一键搞定
  • 修复了 LaunchAgent 重启的 bug

Android（3.8）

  • 砍掉了自更新、后台定位、录屏
  • 前台服务只保留 dataSync
  • 更省电，更干净

开发者相关

模型生态

  • Ollama、vLLM、SGLang：改成 provider-plugin 架构，接入更模块化
  • Kimi Coding：工具调用格式修好了，用原生 Anthropic 格式
  • OpenCode：加了 Go provider，Zen 和 Go 配置合并

记忆系统（3.11）

  • 可选的图片和音频索引
  • 支持 Gemini embedding
  • 自动重新索引

备份功能（3.8）

openclaw backup create openclaw backup verify

可以只备份配置，也可以排除工作区。

升级前注意

3.7 有个 Breaking Change

如果你同时配了 `gateway.auth.token` 和 `gateway.auth.password`，现在要显式指定用哪个：

gateway: auth: mode: token # 或者 password

不然后果：启动失败、配对失败、TUI 进不去。

3.11 改了 Cron 的行为

定时任务不再支持临时 Agent 发送通知，也不支持主会话摘要通知。如果用了这些方式：

openclaw doctor --fix

我的建议

还没升级的：

  • 推荐上 3.12，Dashboard v2 和 Fast Mode 确实好用
  • 升级前跑个备份：`openclaw backup create`
  • 检查 gateway.auth.mode 配置

刚接触 OpenClaw 的：

  • 现在是个好时机，Dashboard v2 上手容易多了
  • 移动端不再是摆设

开发者：

  • Context Engine Plugin 接口（3.7）值得关注，是扩展方向
  • sessions_yield 做复杂编排很有用

最后说两句

OpenClaw 明显在从"极客玩具"往"生产力工具"转：

  1. 产品化：Dashboard v2、移动端、onboarding 流程都在说明这个问题
  2. 企业级：K8s、安全加固、稳定性
  3. 生态：模型越来越多，平台适配越来越全
  4. 社区：每个版本几十个贡献者

这种更新节奏，说明团队在全力冲刺。对用户来说，现在上车正好能赶上产品成熟期的红利。
