# 独立产品人公众号样本验证记录

## 1. 本轮目标

验证以下关键问题：

1. 是否可以通过现成工具搜索到公众号“独立产品人”
2. 是否可以同步出历史文章列表
3. 是否可以抓到文章正文 HTML
4. 正文是否具备稳定转为 Markdown 的基础

---

## 2. 验证环境

- 工具：`wechat-article-exporter`
- 运行方式：Docker 本地运行
- 访问地址：`http://127.0.0.1:3000`
- 登录方式：公众号后台账号扫码登录

---

## 3. 关键验证结果

### 3.1 搜索验证

结果：成功

已成功通过“添加公众号 -> 搜索公众号名称”检索到目标账号：

- 公众号名称：`独立产品人`
- 微信号：`ai_dwdestiny`
- 类型：订阅号

结论：

- 该账号允许通过名称被搜索
- 主工具的“账号发现”链路可用

### 3.2 文章列表同步验证

结果：成功

同步结果：

- 消息总数：`25`
- 已同步消息数：`25`
- 同步进度：`100%`

结论：

- 历史文章列表已可以完整拉取到本地工具数据库
- 本轮至少验证到 25 条历史消息可见

### 3.3 本地数据层验证

结果：成功

从工具本地 IndexedDB 中读取到：

- 数据库名：`exporter.wxdown.online`
- 账号缓存存在
- 文章缓存存在
- 当前同步文章数：`25`

但同时发现：

- `html` 正文缓存数为 `0`

结论：

- 当前状态是“文章列表已同步”
- 但“正文内容尚未批量抓取”
- 所以后续必须显式执行正文抓取步骤，不能误以为同步列表就等于拿到了正文

## 5. 正文抓取验证

### 5.1 直接抓公开链接

结果：失败

从命令行直接访问公开文章链接时，返回的是“环境异常，完成验证后即可继续访问”页面，而不是完整正文。

结论：

- 不能依赖最朴素的 `curl` / `urllib` 直接抓公众号文章正文
- 如果后续脚本不复用浏览器上下文或代理链路，会大量失败

### 5.2 通过工具代理链路抓正文

结果：成功

在浏览器上下文里复用工具同样的代理请求方式后，成功拿到完整文章 HTML。

以第一篇样本为例：

- HTTP 状态：`200`
- HTML 长度：`2779206`
- 包含 `#js_content`：是
- 标题匹配：是
- 不再是“环境异常”页面：是

结论：

- 正文抓取主链路可用
- 后续全量归档必须复用这条抓取方式
